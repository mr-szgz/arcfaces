# Research Summary

- Research Date: `2026-04-03`
- Official Version: Keep a Changelog `1.1.0` (English reference page).
- Key Findings:
  - Changelogs are intended for humans; keep one entry per released version, list newest versions first, and include each release date.
  - Group entries by change type using the standard headings: `Added`, `Changed`, `Deprecated`, `Removed`, `Fixed`, `Security`.
  - Maintain an `Unreleased` section at the top to collect upcoming changes, then move those entries into a new version section at release time.
  - Make version and section headers linkable; note whether the project follows Semantic Versioning.
  - Use ISO 8601 dates (`YYYY-MM-DD`) to avoid ambiguity in release dates.
